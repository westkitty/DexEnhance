# DexEnhance — Roadmap

## Milestone 1: Full-Featured Extension v1

**Goal:** A fully functional Brave extension that enhances both ChatGPT and Gemini with folder organization, smart message queuing, prompt library, and conversation export — distributed privately.

**Success Criteria:**
- Extension loads cleanly on both `chatgpt.com` and `gemini.google.com` with Brave Shields enabled
- All v1 requirements (INFRA, CORE, ADAPT, UI, FEAT, API) pass manual verification
- Packaged as `.zip` for private installation via `brave://extensions`

---

## Phase 1: Foundation & Build Pipeline

**Goal:** Working MV3 extension skeleton that loads without errors in Brave.

**Requirements:** INFRA-01, INFRA-02, INFRA-03, INFRA-04, INFRA-05

**Tasks:**
- Initialize bun project with Vite
- Configure `vite.config.js` with multi-entry MV3 output (background IIFE, content scripts as modules, popup)
- Write `manifest.json` (MV3) with host_permissions, content_scripts, service_worker, icons
- Create full `src/` directory structure with stub files for all modules
- Implement `lib/storage.js` Promise wrapper over `chrome.storage.local`
- Implement `lib/utils.js` (sanitize, truncate, token estimate)
- Validate: `bun run build` succeeds; load unpacked in Brave; no console errors on either site

**UAT Criteria:**
- [ ] `dist/` directory generated by `bun run build` with no errors
- [ ] Extension loads in Brave via "Load unpacked" pointing at `dist/`
- [ ] Visiting `chatgpt.com` and `gemini.google.com` shows no extension errors in console
- [ ] `chrome.storage.local` read/write verified via browser console test call

## Phase 2: Service Worker & Cross-Site Messaging

**Goal:** Reliable cross-site state management through a typed message-passing protocol.

**Requirements:** CORE-01, CORE-02, CORE-03, CORE-04

**Tasks:**
- Implement `background/service_worker.js` with typed message router (`onMessage` listener)
- Define message action constants (`STORAGE_GET`, `STORAGE_SET`, `STORAGE_REMOVE`, etc.)
- Wire background to handle storage operations and respond via `sendResponse`
- Implement `background/api_interceptor.js` skeleton with `updateRules`/`clearRules` stubs
- Write integration test helper: content script sends storage round-trip message, logs result
- Validate: Message round-trip from content script → background → response confirmed in console

**UAT Criteria:**
- [ ] Content script can GET/SET/REMOVE data through the service worker on ChatGPT
- [ ] Same round-trip works on Gemini (cross-domain state sharing confirmed)
- [ ] Background worker survives page navigation without losing state
- [ ] No unhandled promise rejections in background worker console

---

## Phase 3: DOM Adapter Layer

**Goal:** Stable, maintainable DOM interaction abstracted away from per-site selector brittleness.

**Requirements:** ADAPT-01, ADAPT-02, ADAPT-03, ADAPT-04, ADAPT-05

**Tasks:**
- Define `ChatInterface` contract in `content/shared/chat-interface.js`
- Implement `content/gemini/adapter.js` targeting Gemini's current DOM selectors
- Implement `content/chatgpt/adapter.js` targeting ChatGPT's current DOM selectors
- Implement `MutationObserver` for submit button disabled state → `onGeneratingStart`/`onGeneratingEnd`
- Implement `MutationObserver` for chat list container → `onNewChat`
- Log all events to console for validation
- Validate: Both adapters correctly identify and interact with each site's key DOM elements

**UAT Criteria:**
- [ ] `adapter.getTextarea()` returns the active input element on both sites
- [ ] `adapter.getSubmitButton()` returns the send button on both sites
- [ ] `adapter.isGenerating()` returns `true` while AI is responding, `false` otherwise
- [ ] `onGeneratingEnd` fires reliably after AI response completes on both sites
- [ ] `onNewChat` fires when a new conversation appears in the sidebar

## Phase 4: Shadow DOM UI Framework

**Goal:** Injected UI that is visually isolated from host page styles and renders correctly on both sites.

**Requirements:** UI-01, UI-02, UI-03, UI-04, UI-05

**Tasks:**
- Implement `ui/shadow-renderer.js` factory (creates host div, attaches shadow root, injects styles)
- Set up Preact within content script build pipeline; confirm rendering into shadow root
- Implement `ui/styles/theme.css` with `--dex-*` CSS variables; add light/dark detection
- Implement `ui/components/Sidebar.jsx` (collapsible panel, positioned over native sidebar)
- Implement `ui/components/FAB.jsx` (floating button with expandable action menu)
- Validate: Both components visible on both sites; host page CSS does not bleed into shadow root

**UAT Criteria:**
- [ ] Sidebar renders on both ChatGPT and Gemini without visual glitches
- [ ] Host page CSS does not affect shadow DOM component styles (verified via DevTools)
- [ ] Theme CSS variables adapt correctly to each site's light/dark mode
- [ ] FAB renders in correct position; menu items expand on click
- [ ] Sidebar collapse/expand animation works; state persists across page navigation

## Phase 5: Folder & Chat Organization

**Goal:** Users can organize chats into nested folders that persist across sessions and page navigations.

**Requirements:** FEAT-01, FEAT-02, FEAT-03, FEAT-04

**Tasks:**
- Implement folder data model and CRUD operations via service worker message protocol
- Implement `ui/components/FolderTree.jsx` (recursive nested tree, create/rename/delete UI)
- Inject FolderTree into Sidebar; hide native sidebar chat list elements
- Implement chat URL → folder ID mapping with persistence
- Implement trash/soft-delete with restore; Trash view in sidebar
- Validate: Full folder lifecycle and chat organization workflow

**UAT Criteria:**
- [ ] Create, rename, nest, and delete folders; changes persist after browser restart
- [ ] Assign a chat to a folder; chat appears under that folder in the tree
- [ ] Navigate to an assigned chat URL; sidebar correctly highlights the folder
- [ ] Delete a folder → appears in Trash; restore returns it with assigned chats intact
- [ ] Native sidebar list is hidden; only DexEnhance folder tree is visible

## Phase 6: Smart Message Queue

**Goal:** Users can queue multiple messages while the AI is generating; they send automatically in order.

**Requirements:** FEAT-05, FEAT-06, FEAT-07

**Tasks:**
- Implement `content/shared/queue.js` FIFO queue module
- Implement event interception layer (keydown Enter + click intercept → enqueue if generating)
- Implement MutationObserver dequeue trigger (submit re-enabled → dequeue → populate → dispatch)
- Add queue status indicator in sidebar or near input
- Validate: Queue multiple messages, confirm ordered delivery

**UAT Criteria:**
- [ ] While AI is generating, typing and pressing Enter queues the message (not sent immediately)
- [ ] Queue indicator shows number of pending messages
- [ ] After AI response completes, next queued message is automatically sent
- [ ] All queued messages send in FIFO order across a 3+ message queue
- [ ] Pressing Escape or a clear button removes queued messages

## Phase 7: Prompt Library

**Goal:** Users can save, organize, and insert reusable prompts across both sites, with variable substitution.

**Requirements:** FEAT-08, FEAT-09

**Tasks:**
- Implement prompt data model and CRUD via service worker
- Implement `ui/components/PromptLibrary.jsx` (browse/search/insert panel)
- Add `{{variable}}` template parsing and inline fill dialog
- Wire FAB "Prompts" action to open library panel
- Validate: Create on one site, insert on the other with variable filled

**UAT Criteria:**
- [ ] Create a prompt on ChatGPT; open Gemini and find the same prompt in the library
- [ ] Search filters prompts by title and tag
- [ ] Clicking a prompt with no variables inserts it directly into the textarea
- [ ] Clicking a prompt with `{{variables}}` opens a fill dialog; filled text inserts correctly
- [ ] Edit and delete prompts; changes reflect immediately across both sites

## Phase 8: Universal Export

**Goal:** Users can export any conversation as PDF or DOCX with one click.

**Requirements:** FEAT-10, FEAT-11, FEAT-12, FEAT-13

**Tasks:**
- Implement `content/shared/parser.js` conversation DOM extractor for both sites
- Bundle `jspdf` and `docx` libraries (local, no CDN)
- Implement PDF export renderer using parsed conversation data
- Implement DOCX export renderer using parsed conversation data
- Implement `ui/components/ExportDialog.jsx` (format selector, export button)
- Wire FAB "Export" action and sidebar export button to dialog
- Validate: Export full conversations from both sites; verify file contents

**UAT Criteria:**
- [ ] Export parser correctly extracts all user and assistant messages from ChatGPT
- [ ] Export parser correctly extracts all messages from Gemini
- [ ] PDF export opens/renders correctly in macOS Preview
- [ ] DOCX export opens/renders correctly in Pages or Word
- [ ] Export triggered from both FAB menu and sidebar button
- [ ] Blob is generated in content script context; browser download prompt appears natively

## Phase 9: API Interception & Token Display

**Goal:** Token usage and model info surfaced from intercepted API calls; displayed unobtrusively.

**Requirements:** API-01, API-02, API-03

**Tasks:**
- Implement `content/shared/api-bridge.js` with main-world script injection
- Monkey-patch `window.fetch` and `XMLHttpRequest`; extract token/model metadata from response streams
- Bridge data to content script via `window.postMessage` with DexEnhance namespace
- Implement `ui/components/TokenOverlay.jsx` — badge near input area
- Validate: Token count updates after each message on both sites

**UAT Criteria:**
- [ ] Token overlay appears near the input area on both sites
- [ ] Token count updates within 1 second of AI response completing
- [ ] Model name displays correctly (e.g., "GPT-4o", "Gemini 1.5 Pro")
- [ ] Overlay does not interfere with native site UI or input functionality
- [ ] No CORS errors or Brave Shield blocks triggered by the interception mechanism

## Phase 10: Brave Hardening & Private Distribution

**Goal:** Extension is production-ready for private sharing; all features work with Brave Shields UP on a clean profile.

**Requirements:** All (validation sweep)

**Tasks:**
- Audit all network requests; ensure zero external resource loading at runtime
- Verify no remote fonts, no CDN scripts, no external images
- Review Content Security Policy headers for both sites; confirm no violations
- Minimize bundle size; identify and remove unused dependencies
- Package extension as `.zip` (Load Unpacked) and test installation on a clean Brave profile
- Write `README.md` with install instructions for private sharing
- Validate: Full feature walkthrough on clean Brave profile with Shields set to Aggressive

**UAT Criteria:**
- [ ] Brave Shields set to "Aggressive" on both sites — all features still function
- [ ] No network requests to external domains in DevTools Network tab
- [ ] Extension installs from `.zip` on a clean Brave profile with no configuration
- [ ] All 10 phase UAT criteria pass on the clean profile
- [ ] `README.md` install guide tested successfully by a second person
