# Task Templates

When starting a new feature, use this checklist:

- [ ] Clarify requirements.
- [ ] Read relevant existing code.
- [ ] Draft intended changes (create `implementation_plan.md`).
- [ ] Get user approval.
- [ ] Implement changes incrementally.
- [ ] Run `bun run build && bun run test:all` (or relevant `verify` shim).
- [ ] Create `walkthrough.md` documenting the proof of work.
